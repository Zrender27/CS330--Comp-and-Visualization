# CS330--Comp-and-Visualization
Module Eight Reflection
How I Approach Designing Software
When I approach designing software, I first try to understand the goal of the project and what the final product needs to accomplish. For this project, I had to think about the 3D scene as more than just code. I had to consider how each object would be modeled, placed, textured, lit, and viewed by the user. Before writing or changing code, I thought about the objects I wanted to include, how they would fit together visually, and what functions were needed to make the scene work correctly.
This project helped me develop stronger design skills because I had to break a larger scene into smaller parts. Instead of trying to build everything at once, I learned how to focus on individual objects, such as the desk, laptop, notebook, coffee mug, lamp, and other scene elements. I also learned how important placement, scale, rotation, and lighting are in computational graphics. A small change in position or lighting could completely change how the object appeared in the scene.
The design process I followed included planning the scene, identifying the basic shapes needed, applying transformations, adding textures and materials, testing lighting, and making adjustments based on what appeared on the screen. I used an iterative design approach because the scene did not come together perfectly the first time. I had to test, revise, and continue improving the project until the objects looked more complete and organized.
The tactics from this design approach can be applied in future work because most software projects require planning, breaking problems into smaller pieces, testing, and improving over time. Whether I am working on a website, mobile application, database project, or another graphics project, I can use the same process of identifying requirements, designing the structure, building in sections, and revising based on results.
How I Approach Developing Programs
When developing programs, I now approach the work in smaller steps instead of trying to complete everything at once. I start by making sure the basic structure works, then I add more features one at a time. For the 3D scene, this meant first making sure objects rendered correctly, then adding textures, materials, lighting, and camera controls. This helped me find errors more easily because I could usually connect a problem to the most recent change I made.
One new development strategy I used during this project was testing each visual element as I added it. For example, when I added textures or lighting, I had to run the program and check whether the object appeared correctly. If the scene turned black, objects disappeared, or textures did not show, I had to go back through the code and find what caused the issue. This helped me understand the relationship between the code and the visual output.
Iteration was a major part of my development process. I had to revise the scene many times throughout the milestones. Some objects needed to be moved, resized, or rebuilt. Lighting also required several adjustments because the wrong light settings could make the scene too dark or prevent objects from being visible. Each milestone helped me improve part of the project, and the final version was the result of repeated testing and revision.
My approach to developing code has evolved because I am more careful about testing and troubleshooting. At the beginning, I was more likely to feel overwhelmed when errors appeared or when the scene did not render correctly. By the end of the project, I had a better understanding of how to check specific areas of the code, such as transformations, texture loading, shader settings, and lighting functions. This helped me become more confident in debugging and completing the project.

These skills are important for both my education and my future career.
Computational graphics and visualizations gave me new knowledge that I can apply in my future educational pathway. These concepts connect to other areas of computer science because they require logical thinking and an understanding of how code controls output. As I continue my degree, these skills can help me in future courses that involve software design, user interfaces, simulations, or visual data.
Computational graphics and visualizations can also support my future professional pathway. In a professional setting, visualization skills can be useful in software development, web design, game development, simulations, training tools, data visualization, and user experience design. Even if I do not work directly in graphics, this project helped me practice skills that are valuable in many technology roles, such as debugging, organizing code, improving visual presentation, and thinking about how users experience software.
Overall, this project helped me grow as both a designer and a developer. I learned how to plan a visual scene, build it through code, troubleshoot problems, and improve the project through multiple iterations. The experience gave me more confidence in my ability to complete a complex programming project and showed me how computer science can support my long-term academic and professional goals.
